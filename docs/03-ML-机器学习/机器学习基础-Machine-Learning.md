# 机器学习

###  机器学习分类

> **（原图转文字）机器学习分类总览**
>
> **有监督学习**（带标签数据学预测）：线性模型（线性/Lasso/岭/线性判别/逻辑回归）、K近邻、决策树、朴素贝叶斯、支持向量机、神经网络、集成学习（Boosting：AdaBoost/梯度提升树/XGBoost/LightGBM；Bagging：随机森林）。
>
> **无监督学习**（学内在结构）：聚类（K-means、高斯混合、密度/层次/谱聚类）；降维（PCA、SVD、t-SNE、自编码器）。
>
> **半监督学习**：半监督 SVM、图半监督、生成模型、协同训练。
>
> **强化学习**：动态规划、蒙特卡洛树搜索、Q 学习、策略梯度、模仿学习。
>
> **概率模型（横切）**：EM、朴素贝叶斯、贝叶斯网络、HMM、MRF、CRF、MCMC、LDA、VAE。


#### 主要分类

有监督学习：提供数据并提供数据对应结果的机器学习过程    有数据+数据对应标签结果

无监督学习：提供数据但不提供数据对应结果的机器学习过程    只有数据+数据无标注/标签结果

强化学习：通过与环境交互并获取延迟返回进而进行改造行为的学习过程   交互环境，靠延迟奖惩优化行为



### 机器学习处理流程

> **（原图转文字）机器学习处理流程**
>
> 1. **收集数据** → 保证覆盖实际问题的不同方面  
> 2. **数据清洗** → 去除脏数据与不可用样本  
> 3. **特征工程** → 转换/格式化，适配模型输入  
> 4. **选择算法** → 按任务类型（分类/回归/聚类等）与数据特征选型  
> 5. **模型训练** → 用训练集学习规律  
> 6. **模型评估** → 用测试集检验是否达标  
> 7. **模型优化** → 在可用基础上继续提升效果  
> 8. **模型部署** → 上线并实时监控表现


### 特征工程

***\*特征工程\****（Feature Engineering）是机器学习过程中非常重要的一步，指的是通过对原始数据的处理、转换和构造，生成新的特征或选择有效的特征，从而提高模型的性能。简单来说，特征工程是将原始数据转换为可以更好地表示问题的特征形式，帮助模型更好地理解和学习数据中的规律。优秀的特征工程可以显著提高模型的表现；反之，忽视特征工程可能导致模型性能欠佳。

实际上，特征工程是一个迭代过程。特征工程取决于具体情境。它需要大量的数据分析和领域知识。其中的原因在于，特征的有效编码可由所用的模型类型、预测变量与输出之间的关系以及模型要解决的问题来确定。在此基础上，辅以不同类型的数据集（如文本与图像）则可能更适合不同的特征工程技术。因此，要具体说明如何在给定的机器学习算法中最好地实施特征工程可能并非易事。

#### 特征选择

从原始特征中挑选出与目标变量关系最密切的特征，剔除冗余、无关或噪声特征。这样可以减少模型的复杂度、加速训练过程、并减少过拟合的风险。

特征选择不会创建新特征，也不会改变数据结构。

（1）过滤法（Filter Method）

基于统计测试（如卡方检验、相关系数、信息增益等）来评估特征与目标变量之间的关系，选择最相关的特征。

通俗原理：不靠模型，只用统计指标筛特征，优先选和目标关联性高的

（2）包裹法（Wrapper Method）

使用模型（如递归特征消除 RFE）来评估特征的重要性，并根据模型的表现进行特征选择。

通俗原理：套模型反复试组合，按模型效果好坏取舍特征

（3）嵌入法（Embedded Method）

使用模型本身的特征选择机制（如决策树的特征重要性，L1正则化的特征选择）来选择最重要的特征。

通俗原理：训练模型过程中自动筛选，内置机制淘汰无效特征
